# Retail Sales Analysis using SQL (MySQL)

## Project Overview
This project analyzes a large retail transaction dataset using SQL to extract business insights such as revenue trends, customer behavior, product performance, and country-wise sales distribution.

## Tools Used
- MySQL
- SQL

##  Dataset
Retail transactional dataset with 500,000+ records containing:
Invoice details, product information, customer data, and country-level sales.

## Key Analyses Performed

### 1. Revenue Analysis
- Calculated total revenue using Quantity × UnitPrice
- Identified overall business performance

### 2. Monthly Trend Analysis
- Analyzed revenue patterns over time
- Identified seasonal fluctuations

### 3. Product Analysis
- Found top-selling products by quantity
- Identified highest revenue-generating products

### 4. Customer Analysis
- Identified top customers by total spending
- Found revenue concentration among key customers

### 5. Country Analysis
- Analyzed revenue distribution by country
- Identified dominant markets

## Key Insights
- Revenue is highly concentrated in a small number of countries
- Sales show clear seasonal fluctuations across months
- A small percentage of customers generate most of the revenue
- Product demand follows a long-tail distribution

## Skills Demonstrated
- Data analysis using SQL
- Aggregation (SUM, COUNT, GROUP BY)
- Business insight generation
- Data interpretation and reporting

##  Outcome
This project simulates real-world business analytics work and demonstrates the ability to transform raw data into actionable insights.
